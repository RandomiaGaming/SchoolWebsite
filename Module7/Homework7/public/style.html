Styles

Locations

Styles can be defined in many locations. In a separate .css file like what is done for most styles on this website. In a <style> tag like what I used before migration to an external stylesheet. Or they can be defined within the style attribute like what was done for the width and height of the It's free real estate gif on the intrests.html page. Finally in very rare circumstances or on older web pages styles can be defined with certain special use case attributes such as the bgcolor attribute.

Selectors

There are tons of css selectors. Here are a few:

Selector Description
h1 Selects all h1 tags.
.content-container Selects all tags with the content-container class.
#submit-button Selects the element with the id submit-button.
div p Selects all the p tags which are decendants of a div.
div > p Selects all p tags which are children of a div.
h2 + p Selects all p tags which are a sibling of an h2.
input[type="email"] Selects all input tags which have the attribute type="email".
a:hover Selects all a tags which are currently being hovered over.
a::first-line Selects the first line of every a tag.

Box Model

The box model is how CSS and by extension web browsers determine how much space an element should take up on the screen. It is based upon a collection of rectangles each one containing the last. The smallest rectangle is the content of the element. This box contains the text of the element. Next is the padding box, boarder box, and finally encapsulating all the others is the margine box.

Inheritance & Specificity

In css there are times where a different value for a style is defined in different places. In these situations there is a competition to see which style is actually used. Luckily there are rules for which styles take priorety.

  1. Styles marked with !important.
  2. Styles applied with attributes.
  3. Styles applied to an id.
  4. Styles applied to a class.
  5. Styles applied to a tag.

Additionally styles which apply to a child tag always take prioerty over those inherited from the parent.
If two styles are of identical priority the one who was declared furthar down in the document takes priority. Last seen > first seen.
